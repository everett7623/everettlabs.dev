## What it solves

Removing Nezha Agent or Dashboard can leave behind processes, services, cron jobs, Docker containers, and residual files. Nezha Cleaner traces installations and removes them with explicit confirmations.

## Who it is for

System administrators who need to remove Nezha components without leaving orphaned services or containers.

## Why it exists

Uninstalling monitoring agents by hand is risky. Nezha Cleaner applies checks, protects media files, and validates the final state.

## Key capabilities

- Separate Agent and Dashboard cleanup flows
- Process, service, cron, and path detection
- Docker container and image removal
- Docker ownership verification
- Media-file protection
- Post-removal validation

## Engineering notes

Nezha Cleaner is written in Bash and runs on native and Docker-based Nezha installations. Each destructive step asks for confirmation to reduce the chance of accidents.
